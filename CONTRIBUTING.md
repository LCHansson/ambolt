# Contributing to ambolt

## Development setup

```bash
git clone https://github.com/LCHansson/ambolt.git
cd ambolt
```

```r
devtools::install_deps()
devtools::load_all()
```

## Workflow

1. **Load**: `devtools::load_all()` (not `library(ambolt)`) for development
2. **Document**: `devtools::document()` after any change to `R/` — the
   NAMESPACE is auto-generated by roxygen2 and must stay in sync
3. **Test**: `devtools::test()` before committing
4. **Check**: `devtools::check()` before opening a PR

## Adding a new exported function

1. Write the function in the appropriate `R/*.R` file
2. Add roxygen tags: `@param`, `@return`, `@export`, `@examples`
3. If internal (helper): prefix the name with `.` and add `@noRd`
4. Run `devtools::document()` to regenerate `man/` and `NAMESPACE`
5. Add tests in `tests/testthat/`
6. Run `devtools::check()` — target 0 errors, 0 warnings

## Adding a new layout node type

The layout system uses composable R functions that return named lists.
To add a new node type (e.g. a key-value list):

1. Define the R constructor in `R/layout_dsl.R` — call `.drop_nulls()`
2. Export it in the roxygen block (`@export`)
3. Add a dispatch branch in `inst/svelte/components/RenderNode.svelte`
4. Add CSS to the always-emit block in `R/codegen_layout.R` if needed
5. Optionally add a codegen handler in `R/codegen_tree.R` for page-context use

See `plans/specs/modal-dsl.md` "Adding a new node type" for the full
checklist with examples.

## Code conventions

- **No non-ASCII in R code strings** — use `\uxxxx` escapes for Swedish
  characters (e.g. `\u00e5` for `a with ring above`). Comments are OK.
- **Every layout constructor must call `.drop_nulls()`** before returning.
  R's NULL values serialize to truthy empty objects in JavaScript, which
  breaks the frontend's truthiness-based dispatch.
- **Internal helpers**: dot-prefix the function name (`.my_helper`) and
  add `@noRd` to suppress man page generation.
- **Tests**: one file per module in `tests/testthat/`. Regression-lock
  every bug fix. Target: every exported function has at least one test.
